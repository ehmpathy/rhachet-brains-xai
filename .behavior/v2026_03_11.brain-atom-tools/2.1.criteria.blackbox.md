# blackbox criteria: xai brain atom tool use

## usecase.1 = tool invocation with output schema

```
given a brain atom with tools plugged and output schema
  when ask is called with a prompt that requires tool use
    then result.calls.tools contains the tool invocation
      sothat caller can execute the tool and continue
    then result.calls.tools[].slug matches the tool definition slug
      sothat caller knows which tool to invoke
    then result.calls.tools[].input contains parsed arguments
      sothat caller can pass arguments to the tool
    then result.calls.tools[].exid contains provider correlation ID
      sothat caller can correlate responses
    then result.output is null
      sothat caller knows brain deferred to tool

  when ask is called with a prompt the brain can answer directly
    then result.calls is null
      sothat caller knows no tools were requested
    then result.output conforms to the output schema
      sothat caller receives typed structured data
    then result.metrics tracks token usage and cost
      sothat caller can observe resource consumption
```

## usecase.2 = tool invocation without output schema

```
given a brain atom with tools plugged and no response_format sent to provider
  when ask is called with a prompt that requires tool use
    then result.calls.tools contains the tool invocation
      sothat caller can execute the tool
    then result.output is null
      sothat caller knows brain deferred to tool

  when ask is called with a prompt the brain can answer directly
    then result.calls is null
      sothat caller knows no tools were requested
    then result.output contains the response
      sothat caller receives the answer
```

## usecase.3 = multiple tools plugged

```
given a brain atom with multiple tools plugged
  when ask is called and brain requests one tool
    then result.calls.tools has length 1
      sothat caller processes single tool
    then result.calls.tools[0].slug identifies which tool
      sothat caller routes to correct handler

  when ask is called and brain requests multiple tools (parallel)
    then result.calls.tools has length > 1
      sothat caller can process all requested tools
    then each invocation has distinct exid
      sothat caller can correlate results
```

## usecase.4 = error boundaries

```
given a brain atom with tools plugged
  when tool definition has invalid schema
    then error is surfaced at ask time
      sothat caller discovers schema issues early

  when brain returns malformed tool call
    then error includes context about the malformed response
      sothat caller can debug provider issues
```

## usecase.5 = metrics with tools

```
given a brain atom with tools plugged
  when ask completes (with or without tool calls)
    then result.metrics.size.tokens tracks input tokens
      sothat caller knows prompt cost
    then result.metrics.size.tokens tracks output tokens
      sothat caller knows response cost
    then result.metrics.cost.cash tracks monetary cost
      sothat caller can budget usage
    then result.metrics.cost.time tracks elapsed milliseconds
      sothat caller can measure latency
```
