### [1] xAI Function Calling Documentation

> "Function calling enables you to define functions that Grok can invoke by generating structured calls—complete with function names and arguments—embedded in its responses."

### [2] xAI Structured Outputs Documentation
source: https://docs.x.ai/docs/guides/structured-outputs

> "You can combine structured outputs with tool calling to get type-safe responses from tool-augmented queries."

> "Structured outputs with tools is only available for the Grok 4 family of models (e.g., grok-4-1-fast, grok-4-fast, grok-4-1-fast-non-reasoning, grok-4-fast-non-reasoning)."

source: https://docs.x.ai/docs/guides/tools/overview

## best practices

### from xAI docs [1]

1. **tool definition**: include `type: "function"`, `name`, `description`, and `parameters` (JSON Schema)
2. **tool execution loop**: check for `tool_calls` in response, execute locally, append results, continue
3. **parallel calls**: xAI supports parallel function calls by default

### from repo patterns

1. **openai sdk with baseURL**: current `genBrainAtom.ts` uses this pattern (lines 67-72)
2. **zod to JSON schema**: convert via `z.toJSONSchema()` for structured output (line 88)
3. **response_format**: use `json_schema` type for structured output (lines 94-101)

## lessons

1. xAI is openai-compatible — use openai SDK with baseURL override
2. tools go in request body as `tools` array
3. tool calls returned in `response.choices[0].message.tool_calls`
4. each tool call has `id`, `function.name`, `function.arguments` (JSON string)
5. structured output + tools works for grok 4 family models
6. parallel tool calls enabled by default
