# research: external product claims

## claims discovered

### [FACT] xAI API is openai-compatible
source: [xAI Function Calling docs](https://docs.x.ai/docs/guides/function-calling) [1]

> "Function calling enables you to define functions that Grok can invoke"

current codebase confirms: uses openai SDK with baseURL override.

---

### [FACT] parallel tool calls supported by default
source: [xAI Advanced Usage](https://docs.x.ai/docs/guides/tools/advanced-usage) [2]

> "Whether to enable parallel function calling during tool use is available. When true, the model can call multiple functions in parallel."

---

### [FACT] structured outputs + tools works for grok 4 family
source: [xAI Structured Outputs](https://docs.x.ai/docs/guides/structured-outputs) [3]

> "Structured outputs with tools is only available for the Grok 4 family of models"

---

### [FACT] grok 4.1 fast optimized for tool use
source: [xAI Grok 4.1 Fast announcement](https://x.ai/news/grok-4-1-fast) [4]

> "Grok 4.1 Fast is xAI's best tool-calling model with a 2M context window"

> "Through RL training in simulated environments, Grok 4.1 Fast was exposed to a wide variety of tools covering dozens of domains"

---

### [FACT] tool calls not streamed
source: [xAI Function Calling docs](https://docs.x.ai/docs/guides/function-calling) [1]

> "With streaming, the function call is returned in whole in a single chunk, not streamed across chunks"

---

### [SUMP] tool calls priced per invocation
source: [Hackceleration Grok Review](https://hackceleration.com/grok-review/) [5]

> "Tool requests are priced based on two components: token usage and tool invocations"

assumption: pricing model may vary; verify with actual billing.

---

### [KHUE] how does xAI handle malformed tool arguments?
no documentation found on error handling for invalid JSON in `function.arguments`.

question: should we wrap JSON.parse in try/catch and surface helpful error?

---

### [OPIN] xAI lacks streaming with simultaneous tool calls
source: [Hackceleration Grok Review](https://hackceleration.com/grok-review/) [5]

> "The API lacks streaming responses with simultaneous tool calls, a feature GPT-4 offers"

opinion: this is a comparison to competitor; may not affect our usecase.
