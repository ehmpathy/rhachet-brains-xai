# research: external product references

## primary references

### [1] xAI Function Call API
url: https://docs.x.ai/docs/guides/function-calling

establishes:
- tool definition format (`type`, `name`, `description`, `parameters`)
- response format (`tool_calls` array with `id`, `function.name`, `function.arguments`)
- execution loop pattern (check → execute → append → continue)

### [2] xAI Structured Outputs
url: https://docs.x.ai/docs/guides/structured-outputs

establishes:
- `response_format` parameter with `json_schema` type
- grok 4 family required for structured output + tools combo
- schema limitations (no `minLength`, `maxLength`, `allOf`)

### [3] xAI Tools Overview
url: https://docs.x.ai/docs/guides/tools/overview

establishes:
- built-in tools (web search, code execution)
- custom function calls
- tool types taxonomy

### [4] rhachet brains contract
source: rhachet@1.37.14 (`node_modules/rhachet/dist/domain.objects/`)

establishes:
- `BrainAtom` interface with `ask()` method
- `BrainPlugs` with optional `tools: BrainPlugToolDefinition[]`
- `BrainPlugToolDefinition` shape (`slug`, `description`, `schema.input`)
- `BrainPlugToolInvocation` shape (`exid`, `slug`, `input`)
- `BrainOutput` conditional types for output/calls nullability

### [5] openai SDK
source: openai@5.8.2 (used with xai baseURL)

establishes:
- `chat.completions.create()` method
- `tools` parameter format
- `response.choices[0].message.tool_calls` shape

---

## key concepts

| concept | definition | source |
|---------|------------|--------|
| function call | model requests tool execution with structured arguments | [1] |
| tool definition | schema that describes tool name, purpose, parameters | [1] |
| tool invocation | parsed result of model's tool request | [4] |
| structured output | provider-validated JSON response format | [2] |
| parallel tool calls | model requests multiple tools in single response | [1] |
| episode continuation | resume conversation with tool results | [4] |

---

## lessons from references

1. xai mirrors openai API — can reuse SDK knowledge
2. rhachet abstracts provider details — `BrainPlugToolDefinition` is provider-agnostic
3. nullability is typed — rhachet conditional types handle output/calls based on plugs
4. zod to JSON schema conversion is standard — same as structured output
