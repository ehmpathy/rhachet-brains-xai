# research: factory opportunities

## the constraint

**bottleneck**: xAI API latency (~5-10s per call)

if fixed: tests would run faster, tighter feedback loop

**mitigation**: batch assertions via `useThen` to minimize API calls

---

## flow and queues

- **queues**: API response wait time
- **uncontrollable wait**: xAI server response latency
- **WIP limit**: parallel test runs limited by API rate limits

---

## waste

- **handoffs**: none significant
- **repeated setup**: openai client instantiation (could cache, but overhead minimal)
- **idle work**: none
- **manual error-prone**: none — tests are automated

---

## small batches

- **incremental verification**: yes — can test single case first
- **smallest unit**: one tool call scenario
- **early feedback**: yes — run single test file in development

---

## autonomy

- **independence**: yes — all dependencies available
- **consultations**: none needed
- **external blockers**: none

---

## density

- **test execution %**: ~90% wait on API, ~10% actual assertion
- **reclaim time**: use `useThen` to share results across assertions

---

## systems view

**improvement**: batch test assertions to reduce API calls

**scope**: helps this behavior AND all future integration tests

**not a local optimum**: pattern is reusable across repo

---

## recommended opportunity

**use `useThen` pattern consistently** to reduce redundant API calls.

example: one API call, multiple assertions on the result.

this compounds across all integration tests.
