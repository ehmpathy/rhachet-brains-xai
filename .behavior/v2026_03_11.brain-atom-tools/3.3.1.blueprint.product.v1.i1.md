# blueprint: product

## summary

extend `genBrainAtom().ask()` to support tool use via `plugs.tools` parameter:
- accept `BrainPlugToolDefinition[]` via `plugs.tools`
- translate tool definitions to xai format via cast operations
- handle `tool_calls` in response
- enable continuation with `BrainPlugToolExecution[]` as prompt
- return `BrainPlugToolInvocation[]` via `calls.tools`
- extract output and calls independently (not mutually exclusive)

**key design decisions:**
1. **extract cast operations**: explicit boundary between rhachet domain and xai SDK in `src/infra/cast/`
2. **output and calls not mutually exclusive**: model can return both simultaneously
3. **graceful parse error**: throw if parse fails AND no tool calls; silent null if tool calls present

## contracts

### from rhachet (reuse)

- `BrainPlugToolDefinition` — tool declaration
- `BrainPlugToolInvocation` — brain's request to execute
- `BrainPlugToolExecution` — caller's response with result
- `BrainPlugs` — plugs type with tools array
- `BrainOutput` — output with conditional calls/output types

### from openai sdk (xai-compatible)

- `OpenAI.ChatCompletionTool` — tool definition format
- `OpenAI.ChatCompletionMessageToolCall` — tool_calls response format

### from helpful-errors (for signal tests)

- `BadRequestError` — constraint violations (e.g., validation failures)
- `UnexpectedCodePathError` — malfunctions (e.g., service failures)
- `OpenAI.ChatCompletionMessageParam` — message format for continuation

### from this repo (new)

- `castIntoXaiToolDef` — BrainPlugToolDefinition → xai tool
- `castIntoXaiToolMessages` — BrainPlugToolExecution[] → xai messages
- `castFromXaiToolCall` — xai tool_call → BrainPlugToolInvocation

## implementation notes

### independent extraction pattern

```ts
// extract tool calls (always, regardless of content presence)
const toolCalls = response.choices[0]?.message?.tool_calls;
const invocations = toolCalls
  ? toolCalls.map((call) => castFromXaiToolCall({ call }))
  : null;

// extract content (always, regardless of tool_calls presence)
const content = response.choices[0]?.message?.content ?? '';

// parse output (strict mode guarantees schema conformance when content present)
const output =
  content.length === 0
    ? null
    : askInput.schema.output.parse(JSON.parse(content));

// both output AND calls can be present simultaneously
```
