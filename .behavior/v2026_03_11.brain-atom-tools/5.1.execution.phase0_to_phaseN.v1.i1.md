## notes

### key implementation details

1. **tool conversion**: tools converted to openai format on each call
   ```ts
   const tools: OpenAI.ChatCompletionTool[] | undefined =
     askInput.plugs?.tools?.map((tool) => ({
       type: 'function' as const,
       function: {
         name: tool.slug,
         description: tool.description,
         parameters: z.toJSONSchema(tool.schema.input),
       },
     }));
   ```

2. **tool continuation**: for `BrainPlugToolExecution[]` prompts, reconstruct assistant message with tool_calls before tool messages
   ```ts
   if (promptIsToolExecutions) {
     messages.push({
       role: 'assistant' as const,
       content: null,
       tool_calls: executions.map((exec) => ({
         id: exec.exid,
         type: 'function' as const,
         function: {
           name: exec.slug,
           arguments: JSON.stringify(exec.input),
         },
       })),
     });
     for (const execution of executions) {
       messages.push({
         role: 'tool' as const,
         tool_call_id: execution.exid,
         content: JSON.stringify(execution.output),
       });
     }
   }
   ```

3. **response_format always sent**: xai requires response_format even when tools are plugged to ensure structured output when brain answers directly

### issues resolved

- import errors: use `rhachet/brains` instead of `rhachet` for brain types
- type assertions: needed for conditional output/calls types that TypeScript cannot infer
- tool continuation: must include assistant tool_calls message before tool result messages
