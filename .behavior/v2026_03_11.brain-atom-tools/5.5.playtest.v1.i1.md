## detailed verification (optional)

if you want to verify specific behaviors, check the test output for these assertions:

### case4: tool invocation with output schema
- `calls.tools is populated` — brain requested a tool
- `invocation has slug` — tool name is present
- `invocation has input` — arguments are parsed
- `invocation has exid` — correlation ID for continuation
- `output is null when tools called` — brain deferred

### case5: tool invocation without output schema
- `calls.tools is populated` — tool requested
- `output is null` — brain deferred

### case6: brain answers directly
- `calls is null` — no tools requested
- `output is populated` — brain answered

### case7: metrics
- `tokens input is tracked`
- `tokens output is tracked`
- `cost cash is tracked`
- `cost time is tracked`

### case8: tool continuation
- `first ask gets tool call` — initial request
- `second ask with tool result` — continuation works
- `final output is populated` — loop completes
- `calls is null after final answer` — done

### case9-10: error signals
- `brain reasons about the constraint violation`
- `brain reasons about the malfunction`
