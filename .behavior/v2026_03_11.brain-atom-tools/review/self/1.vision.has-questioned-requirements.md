# self-review: 1.vision.has-questioned-requirements

## requirements examined

### 1. support `{ plugs: { tools } }`

| question | answer |
|----------|--------|
| who said? | the wisher, in 0.wish.md |
| why? | rhachet upgraded BrainAtom contract to support tools |
| evidence? | read actual types: BrainPlugs, BrainPlugToolDefinition, BrainOutput from rhachet@1.37.14 |
| what if we didn't? | xai atoms would be behind the rhachet contract; callers couldn't use tools with grok |
| simpler way? | no — this is the contract we must implement |

**verdict**: requirement is valid and necessary.

---

### 2. integration and acceptance tests

| question | answer |
|----------|--------|
| who said? | the wisher: "cover with integration and acceptance tests" |
| why? | verify tools are successfully invocable |
| evidence? | untested code is broken code |
| what if we didn't? | could ship broken tool support |
| simpler way? | could do integration tests only |

**questioned**: do we truly need acceptance tests?

the wish explicitly says "integration and acceptance tests", so this is clearly required. however, the acceptance tests scope is unclear — the vision mentions "full tool loop" but the wish only says "tools are successfully invocable".

**clarification needed**: does "successfully invocable" mean:
- (a) brain requests tools (invocation appears in `calls.tools`) — one-shot
- (b) full loop: ask → tool call → execute → continue → final answer

i assumed (a) for integration tests and left (b) as a question for the wisher.

---

### 3. w/ and w/o output schema

| question | answer |
|----------|--------|
| who said? | the wisher: "w/ and w/o output schema" |
| why? | both are valid use cases |
| evidence? | some callers want structured output with tools, others just want tools |
| what if we didn't? | miss test coverage for valid usage |
| simpler way? | this is already minimal |

**verdict**: requirement is valid. two scenarios: tools with schema, tools without schema.

---

## scope creep identified

the vision document includes **tool continuation** in usecases (step 5 of usage timeline).

the wish does NOT explicitly ask for tool continuation — only that "tools are successfully invocable".

**recommendation**: scope tool continuation out of v1. focus on:
1. brain requests tools (calls.tools populated)
2. works with output schema
3. works without output schema

tool result continuation can be a follow-up.

---

## open questions raised

1. **acceptance vs integration scope**: is "successfully invocable" = invocation appears, or full loop?
2. **tool continuation**: explicitly out of scope for this work?
3. **structured output + tools interaction**: need to verify xai supports both simultaneously

---

## conclusion

the requirements are sound. the main risk is scope creep from tool continuation. recommend we focus on single-round invocation (brain requests tools) for v1, and validate that both with-schema and without-schema work.
