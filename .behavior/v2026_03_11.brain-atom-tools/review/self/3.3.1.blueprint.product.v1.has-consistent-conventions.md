## what i learned from the codebase

### variable names tell a story

reading genBrainAtom.ts, i noticed a clear pattern:

```ts
const config = ...          // noun: the thing
const startedAt = ...       // pastParticiple + prep: when it happened
const systemPrompt = ...    // nounNoun: what kind of prompt
const messages = ...        // noun plural: collection
const jsonSchema = ...      // adjNoun: what format
const response = ...        // noun: API result
const content = ...         // noun: extracted value
const parsed = ...          // pastParticiple: transformed
const output = ...          // noun: final product
const tokensInput = ...     // nounNoun: which tokens
const tokensOutput = ...    // nounNoun: which tokens
const tokensCached = ...    // nounPastParticiple: cached subset
const elapsedMs = ...       // adjUnit: duration
const size = ...            // noun: container
const metrics = ...         // noun: container
```

**insight:** transformed values use past participle (`parsed`, `tokensCached`). untransformed values use nouns (`content`, `response`). containers use simple nouns (`size`, `metrics`).

**why this matters for tools:** if i name tool variables, i should follow this:
- `toolsConverted` not `convertedTools` (nounPastParticiple)
- `invocations` not `toolCalls` (preserve rhachet term)
- `promptParsed` not `parsedPrompt` (nounPastParticiple)

### test organization: one file, multiple cases

the test file has this structure:

```ts
describe('genBrainAtom.integration', () => {
  given('[case1] creation', () => { ... });
  given('[case2] ask is called', () => { ... });
  given('[case3] episode continuation', () => { ... });
});
```

**insight:** the describe block is named after the function. each given block is a scenario. scenarios share the same brainAtom instance at the top.

**why this matters:** if i added a separate test file `genBrainAtom.tools.integration.test.ts`, i would:
1. break the "one file per function" pattern
2. duplicate the brainAtom instantiation
3. fragment coverage of the same function

the correct approach: add cases 4-8 to the extant file.

### SDK exports: get prefix for collections

the SDK file exports:

```ts
export const getBrainAtomsByXAI = (): BrainAtom[] => { ... };
```

**insight:** collection getters use `get` + `Noun` + `By` + `Discriminator` pattern.

**why this matters:** if i were to add a tool-related SDK function (which i'm not), i would use `getToolDefinitionsBy...` not `listTools...`.
