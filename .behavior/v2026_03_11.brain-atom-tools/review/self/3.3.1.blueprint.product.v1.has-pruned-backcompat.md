**backwards compatibility question:**

- did we assume compat "to be safe"? NO — rhachet's conditional type handles this

**key insight:** rhachet's `AsBrainPromptFor<TPlugs>` type definition already handles the conditional. we don't add any backwards compat shim. we just use the rhachet-provided type.

**verdict:** no backwards compatibility added. rhachet's types provide natural narrowing.

### change 3: output becomes nullable

**what the blueprint says:**
```
if tool_calls: extract invocations, set output null
```

**current code (line 171):**
```ts
return new BrainOutput({ output, metrics, episode, series });
```

**what changes:** `output` becomes `TOutput | null` when tools are plugged.

**backwards compatibility question:**
- did wisher request backwards compat? NO — wish is about NEW feature
- is there evidence extant callers would break? MAYBE — callers expect `output: TOutput`
- did we assume compat "to be safe"? NO — rhachet's `AsBrainOutputOutputFor<TPlugs>` handles this

**investigation:** rhachet's BrainOutput generic types use conditional types:
- when `plugs.tools` is NOT provided: `output: TOutput` (guaranteed defined)
- when `plugs.tools` IS provided: `output: TOutput | null` (may be null if tools called)

this is by design in rhachet. callers who don't use tools get `output: TOutput`. callers who use tools get `output: TOutput | null`. the type system forces tool users to handle null.

**verdict:** no backwards compatibility added. rhachet's types provide natural narrowing based on plugs.

## conclusion

**no backwards compatibility was added "to be safe".**

every design choice is evidence-based:
1. rhachet defines the contracts (we implement them, not invent compat shims)
2. the wish doesn't mention backwards compat (it's about NEW feature support)
3. extant callers are unaffected (they don't use tools, their types narrow correctly)

the only changes are:
- reading `plugs.tools` (already typed by rhachet)
- checking prompt type (rhachet types handle)
- returning `calls` and nullable `output` (rhachet types handle)
- adding test cases (additive)

this is a clean feature addition, not a migration requiring backwards compat.
