### summary section

```
when tools are plugged:
- convert tool definitions to openai tools format
- pass to xai API
- parse `tool_calls` from response
- return `calls.tools` with invocations, `output` as null
```

**in wish?** yes. "tools are successfully invocable" requires:
- conversion (xai needs openai format)
- API call (how else would tools be invoked?)
- parse response (to know if tools were called)
- return invocations (caller needs to see them)

**minimum viable?** yes. no extra steps.

**YAGNI verdict:** KEEP

---

```
when prompt is `BrainPlugToolExecution[]` (tool continuation):
- convert executions to openai tool messages
- call xai API with tool results in messages
- parse response and return final output
```

**in wish?** hmm. wish says "tools are successfully invocable". does that include continuation?

**investigation:** the vision added continuation based on rhachet contract (`AsBrainPromptFor<TPlugs>` allows `BrainPlugToolExecution[]`). the wish doesn't explicitly mention it.

**question:** is continuation YAGNI?

**analysis:** if we only do "tools are invocable" without continuation, the feature is incomplete. user gets tool calls but can't feed results back. that's not "successfully invocable" — it's "partially invocable".

**decision:** continuation is implicit in "successfully invocable". a tool invocation isn't successful if you can't complete the loop.

**YAGNI verdict:** KEEP — implicit in wish
