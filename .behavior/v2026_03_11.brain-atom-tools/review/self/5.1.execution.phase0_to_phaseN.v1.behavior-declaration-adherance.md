## summary

implementation matches the spec. one design decision was refined in execution (parse logic) per user feedback.

## key design decisions verification

### 1. extract cast operations at provider boundary

**blueprint:** "explicit boundary between rhachet domain and xai SDK in `src/infra/cast/`"

**implementation:**
- line 82: `askInput.plugs?.tools?.map((tool) => castIntoXaiToolDef({ tool }))`
- line 101: `castIntoXaiToolMessages({ executions })`
- line 130: `toolCalls.map((call) => castFromXaiToolCall({ call }))`

**verdict:** adheres. all provider boundary conversions use dedicated cast operations.

### 2. output and calls not mutually exclusive

**blueprint:** "model can return both simultaneously"

**implementation:**
```ts
// lines 127-131: extract tool calls independently
const toolCalls = response.choices[0]?.message?.tool_calls;
const invocations = toolCalls ? toolCalls.map(...) : null;

// lines 133-140: extract output independently
const content = response.choices[0]?.message?.content ?? '';
const output = content.length === 0 ? null : askInput.schema.output.parse(...);
```

**verdict:** adheres. both extractions are independent, neither checks the other first.

### 3. parse logic (refined from blueprint)

**blueprint:** "throw if parse fails AND no tool calls; silent null if tool calls present"

**implementation:** simplified to just check content length, parse if present

**refinement reason:** user confirmed strict mode (`strict: true` in response_format) guarantees schema conformance when content is present. graceful fallback is unnecessary.

**verdict:** adheres to spirit (fail if parse fails). simplification is correct per strict mode guarantees.

## cast operations verification

### castIntoXaiToolDef

**blueprint:** "maps: slug to name, description, schema.input to parameters (via z.toJSONSchema)"

**implementation (lines 11-20):**
```ts
export const castIntoXaiToolDef = (input: {
  tool: BrainPlugToolDefinition;
}): OpenAI.ChatCompletionTool => ({
  type: 'function' as const,
  function: {
    name: input.tool.slug,
    description: input.tool.description,
    parameters: z.toJSONSchema(input.tool.schema.input),
  },
});
```

**verdict:** adheres exactly.

### castFromXaiToolCall

**blueprint:** "maps: id to exid, function.name to slug, function.arguments to input"

**implementation (lines 10-16):**
```ts
export const castFromXaiToolCall = (input: {
  call: OpenAI.ChatCompletionMessageToolCall;
}): BrainPlugToolInvocation => ({
  exid: input.call.id,
  slug: input.call.function.name,
  input: JSON.parse(input.call.function.arguments),
});
```

**verdict:** adheres exactly.

### castIntoXaiToolMessages

**blueprint:** "produces: assistant message with tool_calls + tool messages with results"

**implementation:** creates assistant message with tool_calls array, then tool messages for each execution.

**verdict:** adheres exactly.
