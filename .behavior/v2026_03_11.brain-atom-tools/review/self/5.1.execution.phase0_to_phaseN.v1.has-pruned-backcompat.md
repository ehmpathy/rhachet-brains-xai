# review: has-pruned-backcompat

## summary

no unnecessary backwards compatibility found.

## review

### ask() signature

the `ask()` signature was extended with optional `plugs` parameter:

```ts
// before
ask(askInput: { prompt: string; ... })

// after
ask(askInput: { prompt: string | BrainPlugToolExecution[]; plugs?: TPlugs; ... })
```

| concern | explicitly requested? | verdict |
|---------|----------------------|---------|
| `plugs` optional | yes - tools are opt-in | holds |
| `prompt` union type | yes - continuation requires executions | holds |

rationale: extant code without `plugs` continues to work. no shims or compatibility layers added.

### output type

output type became conditional based on TPlugs:

```ts
// when tools plugged: output can be null (tool calls instead)
// when no tools: output always present
```

| concern | explicitly requested? | verdict |
|---------|----------------------|---------|
| conditional output | yes - rhachet contract defines this | holds |

rationale: this is the rhachet brain contract, not a compatibility concern.

## conclusion

no backwards-compat hacks found. extension is additive and safe.
