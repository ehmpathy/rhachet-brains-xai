## new test cases needed

### usecase.1: context with creds getter

```ts
given('[caseN] ask with context credentials', () => {
  when('[t0] context provides creds getter', () => {
    const context: ContextBrainSupplier<'xai', BrainSuppliesXai> = {
      'brain.supplier.xai': {
        creds: async () => ({ XAI_API_KEY: process.env.XAI_API_KEY! }),
      },
    };

    const result = useThen('it succeeds', async () =>
      brainAtom.ask({ role: {}, prompt: 'hello', schema: { output } }, context),
    );

    then('response is returned', () => {
      expect(result.output).toBeDefined();
    });
  });
});
```

### usecase.2: getter called fresh per ask

```ts
given('[caseN] getter is called per ask', () => {
  when('[t0] ask is called multiple times', () => {
    let callCount = 0;
    const context: ContextBrainSupplier<'xai', BrainSuppliesXai> = {
      'brain.supplier.xai': {
        creds: async () => {
          callCount++;
          return { XAI_API_KEY: process.env.XAI_API_KEY! };
        },
      },
    };

    const resultFirst = useThen('first ask succeeds', async () =>
      brainAtom.ask({ ... }, context),
    );

    const resultSecond = useThen('second ask succeeds', async () =>
      brainAtom.ask({ ... }, context),
    );

    then('getter was called twice', () => {
      expect(callCount).toEqual(2);
    });
  });
});
```

## test file structure recommendation

consider separate test file for credential supplier tests:

```
src/domain.operations/atom/
  genBrainAtom.ts
  genBrainAtom.integration.test.ts           # extant tests (env var)
  genBrainAtom.credentials.integration.test.ts  # new credential supplier tests
```

this separation allows:
- extant tests to run without modification
- new tests to focus on credential supplier behavior
- clear isolation of concerns
