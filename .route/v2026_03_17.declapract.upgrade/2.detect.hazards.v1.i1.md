## git hazards (found in re-scan)

### 11. gitignore now allows .test*/**/node_modules
**what changed**: `.gitignore` added exceptions:
```
!.test*/**/node_modules
!.test*/**/node_modules/**
```

**why it may break**: test fixture directories that start with `.test` can now commit their `node_modules/`. could accidentally track large dependency trees.

**how to verify**: `git status` — check no unexpected node_modules are staged

### 12. lockfile merge strategy changed to "theirs"
**what changed**: `.gitattributes` added:
```
pnpm-lock.yaml merge=theirs
package-lock.json merge=theirs
```

**why it may break**: on merge conflicts in lock files, git will automatically take the inbound branch's lockfile. could lead to unexpected dependency versions.

**how to verify**: after any merge, run `pnpm install` to regenerate lockfile from package.json
