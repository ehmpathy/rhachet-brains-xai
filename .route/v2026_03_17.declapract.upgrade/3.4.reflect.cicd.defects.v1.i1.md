## analysis

### defect 1: XAI_API_KEY not passed

**classification**: repo quirk

**rationale**: this repo has integration tests that call the xAI API and require `XAI_API_KEY`. the practice correctly introduces the `secrets:` input mechanism for reusable workflows, but repos must wire their specific secrets. this is expected behavior, not a practice bug.

### defect 2: depcheck self-reference

**classification**: repo quirk

**rationale**: this repo includes itself in devDependencies (`"rhachet-brains-xai": "0.2.1"`) to enable acceptance tests that import the published package. this is an uncommon pattern that depcheck does not recognize. an entry in `.depcheckrc.yml` ignores is the correct fix per-repo.

### defect 3: null runtime in slow test report

**classification**: **practice bug**

**rationale**: the "report slow tests" step in the practice uses:

```sh
jq -r '.testResults[] | "\(.name):\(.perfStats.runtime)"' jest-results.json
```

when a test suite is skipped, `perfStats.runtime` is `null`. the jq output becomes the literal string "null", which fails bash arithmetic due to `set -eu`.

**search result**:

```
gh search code --repo ehmpathy/declapract-typescript-ehmpathy "perfStats.runtime" --limit 5

ehmpathy/declapract-typescript-ehmpathy:src/practices/cicd-common/best-practice/.github/workflows/.test.yml: jq -r '.testResults[] | "\(.name):\(.perfStats.runtime)"' ...
```

**search for fix**:

```
gh search code --repo ehmpathy/declapract-typescript-ehmpathy "select.*perfStats.runtime" --limit 5

(no matches)
```

**fix needed in practices**: add `select(.perfStats.runtime != null)` to the jq command:

```sh
jq -r '.testResults[] | select(.perfStats.runtime != null) | "\(.name):\(.perfStats.runtime)"' jest-results.json
```
