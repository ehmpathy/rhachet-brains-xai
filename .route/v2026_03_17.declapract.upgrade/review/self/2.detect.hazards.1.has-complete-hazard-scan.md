## re-scan results

after a second careful pass through the git diff, i found two additional hazards i initially overlooked:

### 11. gitignore now allows .test*/**/node_modules

**what changed**: `.gitignore` added:
```
!.test*/**/node_modules
!.test*/**/node_modules/**
```

**why it may break**: test fixture directories that start with `.test` can now commit their `node_modules/`. this could accidentally track large dependency trees if test fixtures are not careful.

**assessment**: low risk — this is likely intentional for test fixtures that need specific dependencies. verify no unexpected node_modules are staged.

**how it was fixed**: added as hazard #11 in `2.detect.hazards.v1.i1.md` under new "git hazards" section

### 12. lockfile merge strategy changed to "theirs"

**what changed**: `.gitattributes` added:
```
pnpm-lock.yaml merge=theirs
package-lock.json merge=theirs
```

**why it may break**: on merge conflicts in lock files, git will automatically take the inbound branch's lockfile. this could lead to unexpected dependency versions if the merge source has different dependencies than intended.

**assessment**: medium risk — this is a convenience feature but can cause silent dependency changes. developers should run `pnpm install` after every merge to regenerate lockfile from package.json.

**how it was fixed**: added as hazard #12 in `2.detect.hazards.v1.i1.md` under new "git hazards" section
